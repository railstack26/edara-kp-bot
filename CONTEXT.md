# Edara Automation — Контекст проекта для Claude Code

## Обзор системы

Два независимых автоматизированных модуля для логистической компании Edara:

1. **Rate Bot** (Google Apps Script) — мониторит Gmail, обнаруживает запросы на ставку от клиентов, генерирует WeChat-текст для китайских коллег, отправляет в Telegram
2. **KP Bot** (Python / Railway) — получает ответы от китайских коллег в Telegram, переводит, считает ставку, генерирует КП для клиентов

Оба бота работают в одной Telegram группе: **Dream Team** (chat_id: `-5223680248`)

### Инфраструктура

| Компонент | Значение |
|-----------|----------|

### Триггеры
- `checkNewRateRequests` — каждые 10 минут
- `updateCnyRate` — каждый вторник в 9:00 (берёт курс с exchangerate-api.com)

### Логика фильтрации писем
1. Исключить если последнее письмо от нас (`kakamyrat25195`, `kadyro1195`)
2. Исключить если отправитель из `EXCLUDE_SENDERS` (`weishengcn.com`)
3. Исключить если тема содержит слова из `EXCLUDE_SUBJECT` (номера ктк, etd, счет на оплату и т.д.)
4. Исключить если тело содержит фразы из `EXCLUDE_BODY` (прошу взять в работу и т.д.)
5. Если ветка > 2 сообщений — проверить есть ли лейбл `Расчеты`:
   - Есть → 🔄 Обновлённый запрос (обрабатывать с пометкой)
   - Нет → пропустить
6. Обработать если тело содержит триггер из `TRIGGER_BODY` (нужна ставка, прошу рассчитать и т.д.)

### Режимы обработки запросов (SYSTEM_PROMPT)
- **ПРОСТОЙ** — стандартный WeChat для коллег
- **СЛОЖНЫЙ** — EXW/FOR, несколько поставщиков/адресов, консолидация → добавляет блок `=== АНАЛИЗ ЗАПРОСА ===`
- **ОБНОВЛЁННЫЙ** — новое письмо в уже размеченной ветке

### Формат WeChat сообщения
```
[Номер] / [Клиент] / [Тип]
[Город CN] 提货/装箱，到[Город RU на китайском]
[кол-во]件[упаковка если есть]，[вес] KG，[объём] CBM
[Товар на китайском]
HS：[код] — только если известен
Package size: [габариты] — если есть
货好时间：[дата] — если есть
具体装箱地/提货地：[адрес загрузки CN] — для FCL обязательно
```

Упаковка: `托盘`=паллеты, `纸箱`=коробки — сразу после 件 без пробела

### Города RU→CN (для WeChat)

```

[текст]

[вопросы или "Нет вопросов"]
```

## МОДУЛЬ 2 — Edara KP Bot (Python / Railway)

### Расположение
- GitHub: `github.com/railstack26/edara-kp-bot`
- Railway URL: `https://web-production-13b2d.up.railway.app`
- Webhook: `/webhook` (POST)
- Основной файл: `main.py`

### Railway переменные окружения
```
BOT_TOKEN       = <см. Railway Variables → BOT_TOKEN>
OPENROUTER_KEY  = <см. Railway Variables → OPENROUTER_KEY>
CHAT_ID         = -5223680248
CNY_RATE        = 6.9
PORT            = (Railway задаёт автоматически)
```

### LLM модель
`anthropic/claude-3.7-sonnet` через OpenRouter (max_tokens: 1800)

### Архитектура main.py
```
webhook() — принимает POST от Telegram
  ├─ extract_pricing_hints()  — regex извлечение цифр из китайского текста
  ├─ build_messages()         — формирует историю для Claude
  ├─ call_claude()            — вызов OpenRouter API
  └─ send_message()           — отправка ответа в Telegram
```

### extract_pricing_hints() — что извлекает regex
- `pickup_cny` — стоимость забора (提货费 2500元)
- `settlement_rate` — курс CNY платформы (平台结算汇率6.96)
- `usd_per_cbm` — ставка USD/CBM (132 USD/CBM)
- `bill_charge_usd` — сбор за оформление (+USD150/bill)
- `transport_type_hint` — LCL/LTL/FCL

Эти данные передаются Claude как "СТРУКТУРИРОВАННЫЕ ДАННЫЕ ИЗ ТЕКСТА" с пометкой что они обязательны к использованию.

### Ключевые слова активации
Бот реагирует если первая строка начинается с:
- `перевод` — перевести ответ коллег
- `КП` — составить коммерческое предложение

Поддерживается однострочный формат: `КП 1.提货费2500元 2. 长沙-...`

### Диалог с уточнениями
Если данных не хватает → бот отвечает `=== УТОЧНИ ===`
Пользователь отвечает Reply на это сообщение → бот пересчитывает.
Детекция reply: `reply.from.is_bot == true` AND `'=== УТОЧНИ ===' in reply_text`

## Бизнес-логика расчёта ставки (KP Bot)

### LCL и LTL (фиксированная загранставка):
```
Итого = загранставка(USD) + маржа 150 USD + забор(CNY) / курс_CNY
Пример: 2271 + 150 + 1800/6.9 = 2700 USD
```

### LCL/LTL (ставка USD/CBM):
```
Итого = (USD/CBM × расч.CBM) + bill_charge + маржа 150 + забор(CNY)/курс
Пример: 132×13.6 + 150(bill) + 150(маржа) + 2500/6.96 = 2500 USD
```

### FCL:
```
Ставка от коллег УЖЕ включает маржу → передавать как есть, 150 USD НЕ добавлять
```

**Критично:**
- 提货费 ВСЕГДА добавлять отдельно — никогда не "включено в ставку"
- Если коллеги указали курс платформы (平台结算汇率) — использовать его для забора
- Округление: всегда вверх до ближайших 50 или 100

## Шаблоны КП (6 вариантов)

1. **LCL/LTL — один вариант** — `[Имя], По запросу..., Маршрут, ETD, KG/CBM, Ставка, Примечания 1-2`
2. **LCL/LTL — несколько вариантов** — `[Имя], предлагаем N варианта: 1)... 2)..., Примечания 1-2`
3. **FCL — один вариант** — `[Имя], Маршрут, ETD, Ставка USD/тип`
4. **FCL — несколько вариантов** — `[Имя], предлагаем N варианта: 1)... 2)...`
5. **FCL FOB** — с разбивкой FOR + pre carriage + Итого EXW
6. **Отказ** — `[Имя], К сожалению, вынуждены отказать — [причина].`

### Правила примечаний
- **Прим. 1**: если HS-код уже указан в запросе → убрать фразу "Требуется проверка ТНВЭД"
- **Прим. 3**: "Вывоз с Москвы до [город]" — ТОЛЬКО если пункт НЕ Москва
- **Прим. 4**: "Специальная ставка" — ТОЛЬКО если коллеги написали 特价
- **ЗАПРЕЩЕНО**: Болгария, другие страны, СВХ, строка "По запросу [номер]" в теле КП, любые примечания не из шаблонов

## Как работать в Claude Code

### Установка
```bash
npm install -g @anthropic-ai/claude-code
git clone https://github.com/railstack26/edara-kp-bot
cd edara-kp-bot
claude
```

```
Прочитай CONTEXT.md и продолжим работу над Edara KP Bot
```

После правок `main.py` → push в GitHub → Railway задеплоит автоматически (1-2 мин).
Проверка: https://web-production-13b2d.up.railway.app/
Тест: написать в Telegram группу Dream Team: `КП [текст от коллег]`

Открыть Google Apps Script → проект "Edara Rate Bot" → редактировать файл напрямую.
После правок нажать Save (Ctrl+S). Триггеры работают автоматически.
